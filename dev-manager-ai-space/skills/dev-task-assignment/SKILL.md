---
name: 开发任务分配
description: 架构师完成程序功能设计后，可以调用此技能为前/后端开发者分配任务。
---

## 标识符

- **变更ID**（变更级）：`<任务简介>-<YYYYMMDD>-<序号>`，例如 `新增登录功能-20260520-0`。同一次模块变更的前后端任务共享。`<YYYYMMDD>-<序号>` 段与 `architecture/module/<模块>/change-log/<YYYYMMDD>-<序号>.md` 文件名一致。通过看板"关联的变更"列记录，不嵌入任务编号。
- **任务编号**（任务级）：`<任务简介>-<fe|be>`，例如 `新增登录功能-fe`。与变更日志解耦，`-fe`/`-be` 后缀区分前后端。**任务编号 + 关联的变更** 组合作为看板行唯一键。

## 任务粒度

- 以**功能模块的一次变更**为一个任务单元（对应 `architecture/module/<模块>/change-log/` 中一条变更日志）。**不在架构师侧做实现级拆分**（按 API 端点、按页面等），由开发者在各自 Agent 流程中细化。
- 按变更影响面选择分配方式：

| 变更类型 | 分配任务 | 说明 |
|---|---|---|
| 前后端并行（默认） | `-fe` + `-be` | 契约稳定，可并行启动 |
| 仅前端 | 仅 `-fe` | 纯 UI/文案/交互，不涉及 API 或 DB |
| 仅后端 | 仅 `-be` | DB 迁移/性能/索引/内部重构，不影响 API 契约或前端 |
| 前后端串行 | `-fe` + `-be` | 破坏性 API 变更，前端须基于新契约开发；前端任务初始 `blocked`，备注填"前端依赖 `<任务编号-be>` 完成后启动"，后端完成后由用户改为 `todo` |

## 流程

1. **模块名校验**：`Glob /requirements/module/*/`，确认所属模块逐字存在；不存在则停下报告。
2. **读取变更ID**：`Glob /architecture/module/<所属模块>/change-log/*.md`，选取本次变更对应的 change-log 文件（由 architectural-design 阶段刚生成），`Read` 该文件，从首行 `# 变更ID: <变更ID>` 读取变更ID。文件不存在或首行无变更ID 则停下报告：请确认 architectural-design 阶段已生成 change-log 并在首行记录变更ID。**变更ID 不在本技能中编造**。
3. 按任务粒度决策表确定分配哪一侧，为每侧生成任务编号 `<任务简介>-fe` 或 `<任务简介>-be`（`<任务简介>` 取自变更ID 中 `-<YYYYMMDD>-<序号>` 前的段）。
4. 填写看板字段：
   - **任务编号** / **关联的变更**：按"标识符"小节格式。关联的变更 = 步骤 2 读取的变更ID；同次变更前后端取值相同。
   - **所属模块**：与 `requirements/module/` 下文件夹名一致。
   - **描述**：≤20 字简述。
   - **状态**：并行/仅前端/仅后端场景为 `todo`；**串行场景的前端任务填 `blocked`**（等后端完成后由用户改为 `todo`），后端任务填 `todo`。
   - **备注**：串行场景必填（格式见决策表）；其他说明可选，无需重复变更ID。
5. **幂等校验**：`Read` 目标看板，以 任务编号 + 关联的变更 组合键匹配；已存在则跳过并报告。
6. **追加**：前端任务入 `/dev-tasks/frontend.md`，后端任务入 `/dev-tasks/backend.md`。无看板则按 `/templates/开发任务看板模板.md` 新建。仅追加数据行，表头与分隔线不得修改。
7. **回读校验**：`Read` 重新读取，确认新行落入、表头未改、列顺序与模板一致、关联的变更已填。

## 自检清单

全部通过方可报告"任务分配完成"：

- [ ] 变更ID 取自模块 change-log 首行 `# 变更ID: <变更ID>`，未自行编造
- [ ] 任务编号格式 `<任务简介>-<fe|be>`，任务简介取自变更ID 前缀，不含 `-`
- [ ] 关联的变更列填变更ID，前后端同次取值相同
- [ ] 所属模块与 `requirements/module/` 下文件夹逐字一致
- [ ] 描述 ≤20 字
- [ ] 状态：并行/仅前端/仅后端为 `todo`；串行场景前端为 `blocked`、后端为 `todo`
- [ ] 已做幂等校验（组合键不重复）、已回读确认行落入且表头未改
