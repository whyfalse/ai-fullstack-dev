## 强制要求：注释规范

本项目的代码会被 AI 开发者反复读取与修改，注释是 AI 理解项目意图的主要线索，以下为**强制**要求：

- **文件头注释**：每个脚本文件顶部必须有一段文件头注释，说明文件职责、所属模块、对外暴露内容。
- **函数注释**：每个非私有函数必须有职责、参数、返回值、可能异常的说明。
- **关键行注释**：以下场景必须加行内注释——业务规则相关的魔法数字、SQL/索引的特殊写法、事务边界与隔离级别、与前端契约绑定的字段映射、绕过框架默认行为的 hack、临时方案/TODO。
- **注释随代码演进**：修改代码必须同步修改注释，删除代码必须连同注释删除。过时注释比无注释更危险。
- **不写废话注释**：不重复代码已表达的内容，只解释"为什么"和"非显而易见的约束"。

## 易错场景约束

### 分层与契约
- **三层不串味**：控制器只做参数校验与响应组装，业务规则在 Service，数据访问在 Repository/DAO。控制器里写 SQL、Service 里拼响应 DTO 都是分层泄漏。
- **响应裁剪而非透传**：API 响应按前端契约裁剪，不直接返回 DB 实体或表全部字段。实体到响应 DTO 的转换在 Service 或专门 mapper 完成，不在控制器里手抄字段。
- **契约校验在边界**：在控制器入口对请求参数做一次校验，Service/Repository 信任已校验数据，不层层重复校验。

### 数据访问
- **警惕 N+1**：循环里调 DB 必须改为批量查询。列表接口关联字段用 join、in 查询或缓存预加载，不要在循环里逐条取。
- **事务边界最小化**：事务只包必要的数据变更，不包网络调用、远程 IO、长耗时计算。事务越短越好，避免长事务锁竞争。
- **不绕过 schema**：实体字段与项目管理中契约一致，schema 变更必须经架构师确认并走 db-structure-compare 流程，不在代码里用原生 SQL 绕过 ORM 偷偷改字段含义。
- **索引意识**：WHERE、JOIN、ORDER BY 涉及的字段必须有索引支撑；复合索引字段顺序与查询条件匹配。写慢查询前先想索引。

### 状态与并发
- **无状态优先**：服务尽量无状态，会话/缓存状态外置到存储。水平扩展依赖无状态假设。
- **并发安全**：共享资源（缓存、计数器、库存）的读写必须考虑并发——加锁、乐观版本号、原子操作或队列，不要假设"不会并发"。
- **幂等设计**：写接口默认考虑重复调用场景——关键写操作用唯一键、幂等 token 或版本号防重复提交。

### 错误处理
- **失败可见**：异常必须被捕获并显式处理——转成契约定义的错误码返回给前端、打日志告警、或向上传播由统一异常处理接管。禁止 `catch {}` 静默吞异常。
- **不返回原始异常**：对外接口不泄漏堆栈、SQL、表名等内部细节，统一转为契约错误码 + 友好提示。
- **区分业务异常与系统异常**：业务异常（参数错、权限不足、资源不存在）按契约错误码返回；系统异常（DB 连接失败、空指针）走统一兜底，触发告警。

### 安全
- **不入信任**：所有外部输入（请求参数、Header、Body、URL 路径）都不可信。SQL 用参数化查询，不用字符串拼接；反序列化校验类型；文件上传校验类型与大小。
- **鉴权不偷懒**：每个接口显式声明鉴权要求，不依赖"前端会校验"。越权风险（A 用户访问 B 用户资源）必须显式校验资源归属。
- **敏感数据不落地**：密码、密钥、Token 不明文存储、不打日志、不出现在响应里。日志中脱敏手机号、身份证、银行卡等字段。

### 变更范围
- **最小化变更**：修改既有代码只改必须改的部分，不顺手重构。重构应有独立任务。
- **废弃不删除**：暂时不用的接口/方法标记 `@deprecated` 并注明替代方案与计划下线时间，给调用方迁移窗口。
- **保留决策痕迹**：重要的设计决策、绕路方案、临时妥协在注释中记录原因。半年后你自己也记不清为什么这么写。

## 自检清单

提交前对照检查：

- [ ] 文件头、函数、关键行注释齐全且与代码同步
- [ ] 控制器/Service/Repository 职责未串味
- [ ] 响应按契约裁剪，未透传 DB 实体
- [ ] 无 N+1 查询，无循环里调 DB
- [ ] 事务边界最小化，未包网络调用
- [ ] 实体与数据库契约一致，未绕过 schema
- [ ] 写接口幂等，共享资源并发安全
- [ ] 异常显式处理，无静默吞异常，不泄漏内部细节
- [ ] 外部输入已校验，鉴权与越权已处理
- [ ] 敏感数据未明文存储、未打日志、未返回
- [ ] 既有代码只做必要变更，未顺手重构
- [ ] 废弃代码已标记 `@deprecated` 而非删除
