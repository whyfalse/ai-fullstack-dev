## 数据库设计规范

### 设计流程
1. 根据业务实体与关系，画出ER图（逻辑层面）
2. 规范化到第三范式（3NF），再根据查询性能适度反规范化
3. 定义主键、外键、唯一约束、索引
4. 输出DDL语句（SQL或NoSQL schema）

### 命名规范
- 表名：小写复数，下划线分隔：`users`、`order_items`
- 字段名：小写下划线：`user_id`、`created_at`
- 主键统一：`id`（自增整数或UUID）
- 时间戳：`created_at`、`updated_at`（自动维护）

### 字段类型
- 字符串：`VARCHAR` 需指定合理长度，长文本用 `TEXT`
- 数值：明确使用 `INT`、`DECIMAL(10,2)` 等
- 布尔：`BOOLEAN` 或 `TINYINT(1)`
- 时间：`TIMESTAMP` 或 `DATETIME`（带时区用 `TIMESTAMP`）
- 枚举：避免用ENUM类型，用 `VARCHAR` + 应用层校验或关联表

### 索引设计
- 为所有外键、常用查询字段、排序字段创建索引
- 唯一索引用于业务唯一性（如`email`）
- 复合索引遵循最左前缀原则，避免冗余索引

### 扩展性
- 保留 `extra` 或 `metadata` JSON字段存储动态属性
- 软删除：`deleted_at` 字段（默认NULL），查询时过滤

### 示例DDL
```sql
CREATE TABLE users (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP NULL DEFAULT NULL,
  INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
