## 接口定义规范

### 基本要求
- 使用 **RESTful** 风格，复杂场景可结合 **GraphQL**（需说明原因）
- 遵循HTTP方法语义：`GET`（查询）、`POST`（创建）、`PUT`（全量更新）、`PATCH`（部分更新）、`DELETE`（删除）
- 资源命名使用复数名词：`/users`、`/orders/123/items`
- 状态码规范：`200`成功、`201`创建成功、`400`客户端错误、`401`未认证、`403`无权限、`404`资源不存在、`500`服务器错误

### 请求与响应格式
- 请求体：统一使用`application/json`
- 响应体：统一包装结构
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }   // 或数组、null
}
```
- 分页响应：
```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 100
  }
}
```

### 错误处理
- 业务错误码：`code`非0表示错误，具体码值在文档中定义
- 错误详情可附加`details`字段（用于表单字段级错误）

### 文档输出
- 使用 **OpenAPI 3.0** 格式描述所有公开接口
- 每个接口需包含：路径、方法、请求参数（query/body/path）、请求示例、响应示例、错误码说明
